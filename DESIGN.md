# DESIGN — UX / UI

概要
- InstantTest は教員（作成者）と生徒（受験者）の2つの主要ペルソナを想定した軽量なテスト作成／受験アプリです。

ペルソナ
- 教員: クラス管理、テスト作成、問題の編集、AIでの問題自動生成、受験結果の確認
- 生徒: クラスに参加してテストを受験し、終了後に得点や詳細を確認

主要フロー
- 教員: ダッシュボード → クラス作成 → テスト作成（名前/説明/公開/ランダム化）→ 問題を追加／AI生成 → 編集 → 共有（URL/QR）
- 生徒: テスト共有URL または クラス+コードでログイン → 試験開始（exam_session 作成）→ 問題に回答 → 送信 → 集計／レポート閲覧

スクリーン/コンポーネント
- ダッシュボード: クラス一覧、テスト一覧、テストの作成/編集/削除アクション
- テスト編集画面: 問題リスト、問題作成フォーム、選択肢管理、AI生成モーダル（生成→編集→保存）
- AI生成モーダル: 生成中のステータス表示、生成結果の一括編集、選択肢の編集と保存
- 生徒受験ビュー: 1問ずつ／一覧表示どちらも表示可能（現在は一覧で表示→ナビゲート）
- レポート画面: 個人の点数・詳細・試行履歴の参照
- QR／共有モーダル: 共有用URL、QRコード（/api/qr-code を利用）およびコピー機能

アクセシビリティとテーマ
- 高コントラスト用のCSSサンプルが [public/](public/) に含まれています（`teacher_theme_high_contrast.css` 等）。
- フォーム要素はキーボード操作とフォーカス管理を意識して作られています（モーダルにフォーカスを移す等）。

レスポンシブ
- レイアウトはモバイルからデスクトップまで利用可能な幅で設計。モーダルは可変幅で小画面時も操作できるようにしている。

インタラクションの要点
- 削除操作は依存関係がある場合、`cascade=1` を要求して二段階確認を行う（サーバー側でチェック）。
- テストの `randomize` フラグにより、問題順と選択肢順をサーバー側でシャッフル。
- 回答は `student_answers` に保存され、セッション単位で集計できる（`exam_sessions` を使用）。

デザイン改善の提案
- AI生成ワークフローにインクリメンタルなプレビューを追加（大規模テキスト時の応答性改善）。
- モバイル受験時のタイムアウト表示、ネットワークリトライ設計。

備考
- フロントエンドは最小限のReact（DOM APIを直接利用）で構築されているため、コンポーネント化とスタイル整理（CSSモジュールやTailwind等導入）が今後の改善点です。

## 画面再設計メモ（2026-04）

### 目的
- 既存機能を維持したまま、教師は短時間で「作成→配布→確認」まで進める
- 学生は「参加→開始→回答→結果確認」まで迷わず進める
- PC とスマホの両方で情報の優先順位が崩れない

### 情報設計の原則
- 8px グリッドで余白を統一する
- 角丸は 4px から 6px に統一し、業務ツールらしい密度を維持する
- 色はブルー系とグレー系を主軸に 3 から 4 色へ制限する
- 教師画面は一覧性優先、学生画面は集中優先とし、同じ装飾ルールでも見せ方を変える
- 状態変化は派手な演出ではなく、ラベル、ボタン状態、進捗表示で伝える

### 教師用ワイヤーフレーム

```
+--------------------------------------------------------------+
| Header: ダッシュボード / 成績分析 / 学生画面確認             |
+--------------------------------------------------------------+
| Page title        | 今日の状態サマリー 3件                    |
+--------------------------------------------------------------+
| Main                                                   | Side |
|--------------------------------------------------------|------|
| 1. テストを作成                                        |      |
|    - クラス選択                                        | 4.   |
|    - テスト名                                          | クラ |
|    - 公開 / ランダム                                   | ス管 |
|    - 作成ボタン                                        | 理   |
|--------------------------------------------------------|      |
| 2. 配布するテスト一覧                                  |      |
|    - 検索 / クラス絞り込み                             | 5.   |
|    - 公開切替 / ランダム切替                           | 割当 |
|    - 問題編集 / 共有QR / 名称編集 / 削除               |      |
|--------------------------------------------------------|------|
| 3. 状態メッセージ                                      | 補助 |
+--------------------------------------------------------------+
```

#### 教師用の優先順位
- 1位: 新しいテストを最短で作る
- 2位: 既存テストを公開・配布する
- 3位: クラス整理と割り当て変更
- 4位: 分析画面で学習結果を確認する

```
+--------------------------------------------------+
| 結果                                             |
| - 得点                                           |

| - 問題ごとの見直し                               |

```

#### 学生用の優先順位
- 1位: 迷わず開始できること
- 2位: 今どの問題にいるかが常に分かること
- 3位: 回答直後の正誤と解説を静かに提示すること
- 4位: 最後の振り返りで見直すべき点が明確であること

### レイアウト方針
- PC: 教師は 2 カラム、学生は 1 カラム中心で必要時のみ補助列を使う
- スマホ: すべて 1 カラムへ落とし、教師画面は「作成」「一覧」「管理」の順、学生画面は「参加」「開始」「回答」の順に積む
- 重要操作は画面右上やカード右下に固定せず、文脈の直下に置く

### UIトーン
- 参考: Google Classroom、SchoolTakt、Moodle
- 避けるもの: 強いグラデーション、ガラス表現、大きすぎる角丸、不要な影
- 目指す印象: 校務で毎日使える落ち着き、説明を読まなくても進める明快さ、誤操作しにくい距離感
